# junoyi-framework-wework 使用说明

## 1. 模块简介

`junoyi-framework-wework` 提供企业微信相关能力封装，当前支持：

- 企业微信 OAuth 登录相关能力
- 企业微信应用消息发送
  - 文本消息
  - Markdown 消息
  - 模板卡片消息
- 企业微信日程创建与更新
- 企业微信群机器人消息发送
  - 文本消息
  - Markdown 消息
  - 文本消息 @指定人 / @手机号 / @all

## 2. 配置说明

在业务服务的 `application.yml` 中添加配置：

```yaml
junoyi:
  wework:
    enable: true
    corp-id: ww62e2b85ae497924f
    agent-id: 1000002
    secret: your-app-secret
    redirect-uri: https://ythpt.junoyi.com/#/auth/wework/callback
    token-store-type: redis
    redis-key-prefix: junoyi:wework:cp
    group-bot:
      webhooks:
        - https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=webhook-key-1
        - https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=webhook-key-2
```

### 2.1 配置项说明

- `junoyi.wework.enable`：是否启用企业微信能力，默认 `true`
- `junoyi.wework.corp-id`：企业 ID
- `junoyi.wework.agent-id`：企业应用 AgentId
- `junoyi.wework.secret`：企业应用 Secret
- `junoyi.wework.redirect-uri`：OAuth 回调地址
- `junoyi.wework.token-store-type`：token 存储方式，支持 `memory` 和 `redis`
- `junoyi.wework.redis-key-prefix`：Redis 存储前缀
- `junoyi.wework.group-bot.webhooks`：企业微信群机器人 webhook 列表，可配置一个或多个

### 2.2 Redis 说明

当 `token-store-type=redis` 时，需要项目中存在 `RedissonClient`。
如果未提供 `RedissonClient`，启动时会抛出异常。

## 3. 自动装配说明

模块启用后会自动装配以下核心 Bean：

- `WeWorkClient`
- `WeWorkAppMessageHelper`
- `WeWorkScheduleHelper`
- `WeWorkGroupBotHelper`
- `RestTemplate`

业务代码中可直接通过 Spring 注入使用。

## 4. 企业微信应用消息

### 4.1 注入 Helper

```java
@Resource
private WeWorkAppMessageHelper weWorkAppMessageHelper;
```

### 4.2 发送文本消息

```java
weWorkAppMessageHelper.sendText("zhangsan", "这是一条文本消息");
```

### 4.3 发送 Markdown 消息

```java
weWorkAppMessageHelper.sendMarkdown("zhangsan", "### 这是一条 Markdown 消息");
```

### 4.4 发送模板卡片消息

```java
NoticeTemplateCardMessage card = new NoticeTemplateCardMessage();
card.setTouser("zhangsan");
card.setSourceDesc("任务中心");
card.setMainTitle("任务通知");
card.setMainDesc("你有一条新的任务待处理");
card.setSubTitle("合同审批任务");
card.setEmphasisTitle("截止时间");
card.setEmphasisDesc("2025-08-10 18:00:00");
card.setQuoteTitle("任务备注");
card.setQuoteText("请及时处理");
card.setJumpUrl("https://ythpt.junoyi.com/#/task/detail/1");
card.setJumpTitle("查看任务");
weWorkAppMessageHelper.sendCard(card);
```

## 5. 企业微信日程

### 5.1 注入 Helper

```java
@Resource
private WeWorkScheduleHelper weWorkScheduleHelper;
```

### 5.2 创建日程

```java
WeWorkScheduleRequest request = new WeWorkScheduleRequest();
request.setSummary("任务标题");
request.setDescription("任务描述");
request.setOrganizer("zhangsan");
request.setAttendees(List.of("zhangsan", "lisi"));
request.setRemarks("任务备注");
request.setStartTime(1754800800L);
request.setEndTime(1754808000L);
String scheduleId = weWorkScheduleHelper.createSchedule(request);
```

### 5.3 更新日程

```java
weWorkScheduleHelper.updateSchedule(scheduleId, request);
```

## 6. 企业微信群机器人消息

### 6.1 注入 Helper

```java
@Resource
private WeWorkGroupBotHelper weWorkGroupBotHelper;
```

### 6.2 发送到全部已配置 webhook

```java
weWorkGroupBotHelper.sendText("这是一条群机器人文本消息");
weWorkGroupBotHelper.sendMarkdown("### 这是一条群机器人 Markdown 消息");
```

### 6.3 发送到指定 webhook

```java
String webhook = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxx";
weWorkGroupBotHelper.sendText(webhook, "指定 webhook 文本消息");
weWorkGroupBotHelper.sendMarkdown(webhook, "### 指定 webhook Markdown 消息");
```

### 6.4 发送带 @ 的文本消息

```java
weWorkGroupBotHelper.sendText("请尽快处理任务", List.of("zhangsan"), null);
weWorkGroupBotHelper.sendText("请相关同学处理", null, List.of("13800000000"));
weWorkGroupBotHelper.sendText("全员通知", List.of("@all"), null);
```

### 6.5 发送到指定 webhook 并携带 @

```java
weWorkGroupBotHelper.sendText(
    webhook,
    "任务已驳回，请相关人员尽快处理",
    List.of("zhangsan", "lisi"),
    null
);
```

## 7. 常见说明

### 7.1 应用消息与群机器人消息的区别

- `WeWorkAppMessageHelper`：给企业微信应用用户发消息，依赖 `corp-id / agent-id / secret`
- `WeWorkGroupBotHelper`：给企业微信群机器人发消息，依赖 webhook

### 7.2 Markdown @ 说明

企业微信群机器人的结构化 `@` 主要用于文本消息。
如果既要提醒指定人，又要展示复杂内容，建议：

1. 先发一条带 `@` 的文本消息
2. 再发一条 Markdown 消息展示详情

### 7.3 webhook 多配置说明

当配置多个 `group-bot.webhooks` 时：

- `sendText(String content)` 会发送到全部 webhook
- `sendMarkdown(String content)` 会发送到全部 webhook
- 指定 webhook 的重载方法只会发到当前传入地址
